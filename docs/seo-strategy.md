# SEO・集客戦略（2026年2月策定）

## 現状の課題

- Google Search Console: インデックス登録が1ページのみ
- 「代替ページ（適切な canonical タグあり）」10件 → プリレンダリングで対応済み（2026-02-07）
- sitemap に17ページしかない
- OG 画像が全ページ共通で SNS のクリック率が低い
- ソーシャルシェアが少ない

## 優先施策

### 1. Programmatic SEO（sitemap 拡張 + プリレンダリング拡大）

**効果: 最大 / 工数: 小 / コスト: 0円**

現在17ページ → 目標: 300〜500ページ

#### 生成パターン

**a. 年代 × 通貨 × キリの良い金額**
- 主要年代: 1950, 1955, 1960, 1965, ... 2020（5年刻み）
- 4通貨: jpy, usd, gbp, eur
- 金額: 1, 10, 100, 1000, 10000, 100000, 1000000（通貨に応じて）

**b. 歴史的イベント年 × 代表的な金額**

| 年 | イベント | 代表的な金額 |
|---|---|---|
| 1945 | 終戦 | jpy/100 |
| 1964 | 東京オリンピック | jpy/100, jpy/1000 |
| 1971 | ニクソンショック | usd/1, usd/100 |
| 1973 | オイルショック | jpy/1000, usd/100 |
| 1985 | プラザ合意 | usd/1, usd/100, jpy/10000 |
| 1989 | バブル絶頂期 | jpy/10000, jpy/1000000 |
| 1995 | 阪神大震災 | jpy/10000 |
| 1997 | アジア通貨危機 | jpy/10000 |
| 2000 | IT バブル | usd/100, eur/100 |
| 2008 | リーマンショック | usd/100, jpy/10000 |
| 2011 | 東日本大震災 | jpy/10000 |
| 2020 | コロナショック | jpy/10000, usd/100 |

#### 注意点

- Thin content にならないよう、各ページに年代ごとの歴史的コンテキストを含める
- SEOContent.tsx を拡張して年代固有の解説を表示
- Google の Helpful Content 基準を意識

#### 実装方法

1. `scripts/generate-sitemap.cjs` を作成（ルート定義 + sitemap.xml 生成）
2. `scripts/prerender.cjs` が既に sitemap からルートを読み取る仕組みがあるので、sitemap を拡張するだけ

### 2. 動的 OG 画像（Cloudflare Workers）

**効果: 大 / 工数: 中 / コスト: 0円**

#### 概要

計算結果ごとに固有の OG 画像を生成。SNS でシェアされた時に結果が一目でわかる。

#### 技術スタック

- Cloudflare Workers + `workers-og`（Satori + resvg）
- Workers 無料プラン: 100,000 リクエスト/日
- 日本語フォント: Noto Sans JP サブセットを KV に格納

#### URL 設計

`https://imaikura.creco.net/og/{year}/{currency}/{amount}.png`

#### 必要な変更

- Workers プロジェクト作成
- `prerender.cjs` の OG 画像 URL を動的パスに変更
- `useSEO.ts` の ogImage を動的 URL に変更

### 3. Bluesky Bot
